# Learn Vue with ChatGPT (Basic)

## Overview

This course is a comprehensive, hands-on, lesson-by-lesson curriculum to take you from zero (or intermediate) to advanced productivity with Vue 3. Each lesson contains goals, theory, examples, exercises, project tasks, debugging notes, and quick-check quizzes with answers. Where appropriate, both Composition API (recommended) and brief Options API equivalents are shown.

## Quick Learning Path

### Lesson 1 — Core Concepts

1. The Vue Instance / App

   ```js
   import { createApp } from "vue";
   import App from "./App.vue";

   createApp(App).mount("#app");
   ```

   Note: Equivalent to ReactDOM.createRoot

2. Templates

   - v-if
   - v-else / v-else-if
   - v-for
   - v-bind (: shorthand)
   - v-on (@ shorthand)
   - v-model (for two-way binding)

   ```js
   <div v-if="loggedIn">Hello, {{ user.name }}</div>
   <div v-else>Please log in.</div>

   <img :src="imageUrl" />
   <ul>
     <li v-for="todo in todos" :key="todo.id">{{ todo.text }}</li>
   </ul>
   <input v-model="userInput" />
   <button @click="handleClick">Click Me</button>
   ```

3. Reactivity System

   - `ref` for primitive values
   - `reactive` for objects and arrays
   - `computed` for derived state
   - `watch` for side effects

   ```js
   import { ref, reactive, computed, watch } from "vue";
   const count = ref(0);
   const state = reactive({ name: "Vue", version: 3 });
   const doubled = computed(() => count.value * 2);
   watch(count, (newVal, oldVal) => {
     console.log(`Count changed from ${oldVal} to ${newVal}`);
   });
   ```

4. Composition API

   - `setup` function
   - Lifecycle hooks: `onMounted`, `onUnmounted`, etc.

   ```js
   import { defineComponent, ref, onMounted } from "vue";

   export default defineComponent({
     setup() {
       const message = ref("Hello, Vue!");

       onMounted(() => {
         console.log("Component mounted");
       });

       return { message };
     },
   });
   ```

### Lesson 2 - Essentials

1. Components Basics

   - Defining components
   - Props and Emits
   - Slots

   ```vue
   <!-- ChildComponent.vue -->
   <template>
     <div>
       <slot></slot>
       <p>{{ message }}</p>
     </div>
   </template>

   <script>
   export default {
     props: {
       message: String,
     },
   };
   </script>
   ```

   ```vue
   <!-- ParentComponent.vue -->
   <template>
     <ChildComponent message="Hello from Parent!">
       <h1>Title from Parent</h1>
     </ChildComponent>
   </template>

   <script>
   import ChildComponent from "./ChildComponent.vue";

   export default {
     components: {
       ChildComponent,
     },
   };
   </script>
   ```

2. Conditional Rendering and List Rendering

   - `v-if`, `v-else`, `v-else-if`
   - `v-for`

   ```vue
   <template>
     <div v-if="isLoggedIn">Welcome back!</div>
     <div v-else>Please log in.</div>

     <ul>
       <li v-for="item in items" :key="item.id">{{ item.name }}</li>
     </ul>
   </template>

   <script>
   export default {
     data() {
       return {
         isLoggedIn: false,
         items: [
           { id: 1, name: "Item 1" },
           { id: 2, name: "Item 2" },
         ],
       };
     },
   };
   </script>
   ```

3. Two-Way Binding

   - `v-model` (Binding input values to data properties)

   ```vue
   <template>
     <input v-model="username" placeholder="Enter your name" />
     <p>Your name is: {{ username }}</p>
   </template>

   <script>
   export default {
     data() {
       return {
         username: "",
       };
     },
   };
   </script>
   ```

4. Events

   - `v-on` or `@` for event handling

   ```vue
   <template>
     <button @click="incrementCount">Clicked {{ count }} times</button>
   </template>

   <script>
   export default {
     data() {
       return {
         count: 0,
       };
     },
     methods: {
       incrementCount() {
         this.count++;
       },
     },
   };
   </script>
   ```

### Lesson 3 - State Management

1. Composables

   - Creating reusable logic with composables

   ```js
   // useCounter.js
   import { ref } from "vue";

   export function useCounter() {
     const count = ref(0);
     const increment = () => {
       count.value++;
     };
     return { count, increment };
   }
   ```

   Note: Like React hooks (useState)

   ```vue
   <template>
     <button @click="increment">Count: {{ count }}</button>
   </template>

   <script>
   import { useCounter } from "./useCounter";

   export default {
     setup() {
       const { count, increment } = useCounter();
       return { count, increment };
     },
   };
   </script>
   ```

2. Pinia (Vuex replacement)

   - Setting up Pinia for state management

   ```js
   // store.js
   import { defineStore } from "pinia";

   export const useMainStore = defineStore("main", {
     state: () => ({
       count: 0,
     }),
     actions: {
       increment() {
         this.count++;
       },
     },
   });
   ```

   ```vue
   <template>
     <button @click="increment">Count: {{ count }}</button>
   </template>

   <script>
   import { useMainStore } from "./store";

   export default {
     setup() {
       const mainStore = useMainStore();
       return {
         count: mainStore.count,
         increment: mainStore.increment,
       };
     },
   };
   </script>
   ```

### Lesson 4 - Routing and Fetching data

1. Vue Router Basics

   - Setting up routes and navigation

   ```js
   // router.js
   import { createRouter, createWebHistory } from "vue-router";
   import Home from "./pages/Home.vue";
   import About from "./pages/About.vue";
   import Contact from "./pages/Contact.vue";

   export default createRouter({
     history: createWebHistory(),
     routes: [
       { path: "/", component: Home },
       { path: "/about", component: About },
       { path: "/contact", component: Contact },
     ],
   });
   ```

2. Navigation Links

   - Using `<router-link>` for navigation

   ```vue
   <template>
     <nav>
       <router-link to="/">Home</router-link>
       <router-link to="/about">About</router-link>
       <router-link to="/contact">Contact</router-link>
     </nav>
     <router-view></router-view>
   </template>
   ```

3. Fetching Data

   - Using `fetch` or `axios` to get data from APIs

   ```vue
   <template>
     <div>
       <h1>Users</h1>
       <ul>
         <li v-for="user in users" :key="user.id">{{ user.name }}</li>
       </ul>
     </div>
   </template>

   <script>
   import { ref, onMounted } from "vue";

   export default {
     setup() {
       const users = ref([]);

       onMounted(async () => {
         const response = await fetch(
           "https://jsonplaceholder.typicode.com/users"
         );
         users.value = await response.json();
       });

       return { users };
     },
   };
   </script>
   ```

## Zero to Advanced Learning Path

### Lesson 1 — What is Vue.js

progressive, incrementally adoptable JavaScript framework used to build user interfaces and single-page applications (SPAs)

1. **Core Principles**

   a. **Declarative Rendering**: You describe _what_ the UI should look like, and Vue handles _how_ it updates.
   Example:

   ```html
   <div id="app">{{ message }}</div>
   ```

   Note: _Vue automatically updates the DOM when data changes_

   b. **Component-Based Architecture**: Build encapsulated components that manage their own state and logic. UI is built from small, isolated, reusable components, i.e. Header, Sidebar, Card, Button, Pages etc. Each component has its own: **Template**, **Script**, **Styles**.

   c. **Reactive Data System**: Vue tracks data dependencies via a proxy-based reactivity system. When reactive data changes, the UI updates _automatically_. This allows you to focus on _logic_, not manual DOM manipulation.

2. **Vue Ecosystem**

   Vue is not just a framework. It is an ecosystem of tools:
   | **Package** | **Purpose** |
   |----------|----------|
   | **Vue Core** | _The runtime framework_ |
   | **Vue Router** | _SPA navigation_ |
   | **Pinia** | _State management_ |
   | **Vite** | _Development server + bundler_ |
   | **DevTools** | _Inspector inside browser_ |
   | **SFC** | _Compiler .vue file compiler_ |

3. **Creating a Vue App**
   Using CDN:

   ```html
   <script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

   <div id="app">
     <h1>{{ message }}</h1>
   </div>

   <script>
     const { createApp } = Vue;
     createApp({
       data() {
         return {
           message: "Hello, Vue!",
         };
       },
     }).mount("#app");
   </script>
   ```

   Explanation:

   - `createApp()` creates a Vue instance.
   - `data()` returns the reactive properties.
   - `{{ message }}` binds to DOM.
   - Updating `message` updates UI instantly.

4. **Vue Directives (Foundation of Vue Templates)**

   Vue directives are special attributes prefixed with `v-` that provide reactive behavior to the DOM.

   a. **v-bind**: Dynamically bind attributes to expressions.

   ```html
   <img v-bind:src="imageUrl" />
   <!-- Shorthand -->
   <img :src="imageUrl" />
   ```

   b. **v-model**: Two-way data binding for form `input`s.

   ```html
   <input v-model="username" placeholder="Enter your name" />
   <p>Your name is: {{ username }}</p>
   ```

   c. **v-if / v-else-if / v-else**: Conditional rendering.

   ```html
   <div v-if="isLoggedIn">Welcome back!</div>
   <div v-else>Please log in.</div>
   ```

   d. **v-for**: Render lists based on an array.

   ```html
   <ul>
     <li v-for="item in items" :key="item.id">{{ item.name }}</li>
   </ul>
   ```

   e. **v-on**: Event handling.

   ```html
   <button v-on:click="increment">Click me</button>
   <!-- Shorthand -->
   <button @click="increment">Click me</button>
   ```

   f. **v-show**: Toggle element visibility via CSS `display`.

   ```html
   <div v-show="isVisible">This is visible</div>
   ```

   g. **v-pre**: Skip compilation for this element and its children.

   ```html
   <div v-pre>{{ rawContent }}</div>
   ```

   h. **v-cloak**: Keep element hidden until Vue instance is ready.

   ```html
   <div v-cloak>{{ message }}</div>
   ```

   i. **v-once**: Render element/component only once.

   ```html
   <div v-once>{{ message }}</div>
   ```

5. **Styling in Vue**

   a. Inline Style Binding

   ```html
   <div :style="{ color: activeColor, fontSize: fontSize + 'px' }">
     Styled Text
   </div>
   ```

   b. Class Binding

   ```html
   <div :class="{ active: isActive, 'text-large': isLargeText }">
     Class Bound Text
   </div>
   ```

   c. Scoped Styles in Single File Components (SFC)

   ```vue
   <template>
     <div class="scoped-text">This text is scoped</div>
   </template>
   <style scoped>
   .scoped-text {
     color: blue;
   }
   </style>
   ```

   Note: Scoped styles apply only to this component.

6. **Common Errors & Fixes**

   - Error: "Property not defined on instance"
     - **Cause**: Trying to access a property that is not defined in the component's data or setup function.
     - **Fix**: Ensure property is defined in `data()` or `setup()`.
   - Error: Template Syntax Error
     - **Cause**: Invalid syntax in template (e.g., missing closing tags).
     - **Fix**: Check template for syntax errors.
     - Also, **Vue templates allow only one root element**.

7. **Mini Project — Interactive Counter**

   Create a simple counter app with increment and decrement buttons.

   ```vue
   <template>
     <div class="counter-app">
       <h1>Counter: {{ count }}</h1>
       <button @click="decrement">-</button>
       <button @click="increment">+</button>
     </div>
   </template>

   <script>
   import { ref } from "vue";

   export default {
     setup() {
       const count = ref(0);

       const increment = () => {
         count.value++;
       };

       const decrement = () => {
         count.value--;
       };

       return { count, increment, decrement };
     },
   };
   </script>

   <style scoped>
   .counter-app {
     text-align: center;
     margin-top: 50px;
   }
   button {
     font-size: 20px;
     margin: 5px;
     padding: 10px 20px;
   }
   </style>
   ```

   Explanation:

   - `ref(0)` creates a reactive `count` variable initialized to `0`.
   - `increment` and `decrement` functions modify `count`.
   - Buttons call these functions on click.
   - The UI updates automatically when `count` changes.
