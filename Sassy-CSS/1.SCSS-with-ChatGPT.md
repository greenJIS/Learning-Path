# Sassy CSS (SCSS)

SCSS (Sassy CSS) is an extension of CSS that adds features like:

- Variables
- Nesting
- Functions & Mixins
- Conditionals & Loops
- Imports (partials)
- Built-in utility functions (color manipulation, maps, lists, etc.)
- A proper architecture (modular CSS)

Note: _SCSS compiles into normal CSS, so browsers only see CSS_

## Lesson 1 - SCSS Basics [⬆](#)

### Install SCSS Compiler

Install Sass Globally: `npm install -g sass`

Or local project: `npm install sass`

Compile your file: `sass input.scss output.css --watch`

### SCSS Basic Syntax

SCSS is a superset of CSS—everything you already know works.

`style.scss`:

```scss
body {
  background: #f5f5f5;
}
```

This compiles directly to CSS.

### Variables

This is the most commonly used SCSS feature.

```scss
$primary: #b46aff;
$padding-base: 1rem;

button {
  background: $primary;
  padding: $padding-base 1.5rem;
}
```

Variables can store:

- Colors
- Fonts
- Sizes
- Themes
- Z-index
- Durations

### Nesting (Hierarchical CSS)

<div style="display: flex; gap: 1rem; align-items:center;">
  <div> Traditional CSS:

```css
.navbar {
  background: white;
}
.navbar a {
  color: black;
}
.navbar a:hover {
  color: red;

```

  <div>⇒</div>
  <div>SCSS version:

```scss
.navbar {
  background: white;

  a {
    color: black;

    &:hover {
      color: red;
    }
  }

```

  </div>

Goal: Clean, readable & structured.

### Partials & Imports (Project Structure)

SCSS lets you split code into files.

Example structure:

```
scss/
  _variables.scss
  _buttons.scss
  _navbar.scss
  style.scss
```

Underscore means "partial" (not compiled alone).
Imports combine them:

`style.scss`:

```scss
@import "variables";
@import "buttons";
@import "navbar";
```

This compiles into one final CSS file.

### Mixins (Reusable Style Functions)

Example without mixin:

```scss
.box {
  transition: all 0.3s;
}
.card {
  transition: all 0.3s;
}
```

With mixin:

```scss
@mixin smooth {
  transition: all 0.3s;
}

.box {
  @include smooth;
}

.card {
  @include smooth;
}
```

Mixins can also accept parameters:

```scss
@mixin size($w, $h) {
  width: $w;
  height: $h;
}

.avatar {
  @include size(40px, 40px);
}
```

### Extend (Inheritance)

Use it when many selectors share identical styles.

```scss
%btn-base {
  padding: 0.7rem 1.3rem;
  border-radius: 0.5rem;
}

.btn-primary {
  @extend %btn-base;
  background: #b46aff;
}

.btn-secondary {
  @extend %btn-base;
  background: #6c757d;
}
```

`%placeholder` selectors compile only when extended.

### Built-in Color Functions

SCSS gives you powerful color manipulation:

```scss
$primary: #b46aff;

.btn {
  background: $primary;
  &:hover {
    background: darken($primary, 10%);
  }
}
```

Useful functions:

- `darken()`
- `lighten()`
- `mix()`
- `rgba()`
- `adjust-hue()`
- `saturate()` / `desaturate()`

### Operators (Math in SCSS)

```scss
$card-gap: 20px;
$card-width: 200px;

.container {
  width: $card-width * 3 + $card-gap * 2;
}
```

This makes SCSS excellent for layout calculations.

### Maps (Key-Value Data Structures)

Important for theming (Bootstrap uses maps heavily).

```scss
$colors: (
  "primary": #b46aff,
  "accent": #ff72d0,
);

.button {
  background: map-get($colors, "accent");
}
```

This is how you define design systems.

### Control Flow (If / Each / For)

Example: Loop to generate spacing utilities:

```scss
@for $i from 1 through 5 {
  .m-#{$i} {
    margin: #{$i}rem;
  }
}
```

Bootstrap itself uses heavy looping like this.

### When to Use SCSS in Real Projects

Use SCSS when you need:

- A **design system** (colors, spacing, radius, fonts)
- **Modular architecture**
- **Reusable UI components**
- **Custom themes**
- **Bootstrap or Tailwind modification**
- **Responsive logic**
- **Cleaner, more maintainable code**

## Lesson 2 — SCSS Architecturess [⬆](#)

### Why Architecture Matters

SCSS gives power, but without structure your project becomes:

- Hard to maintain
- Hard to scale
- Hard to debug
- Easy to break

Architecture ensures:

- Modularity
- Clear separation of responsibility
- Small, readable files
- Reusable components
- Predictable theme system

### Standard SCSS Folder Structure (7–1 Architecture)

```sh
scss/
│
├── abstracts/  # mixins, variables, functions
├── vendors/    # external libs (bootstrap, normalize)
├── base/       # resets, typography, global
├── layout/     # grid, header, footer, nav, sidebar
├── components/ # buttons, cards, modals, forms
├── pages/      # page-specific styles
├── themes/     # light, dark, brand themes
│
└── main.scss   # main entry file
```

This structure is the industry standard (7-1 architechture).

#### Explanation:

⇒ **`abstracts/`**

_These never output CSS directly._

Contains:

- `_variables.scss`
- `_mixins.scss`
- `_functions.scss`
- `_breakpoints.scss` (optional)
- `_colors.scss` (if separated)

Example:

```scss
$primary: #b46aff;
$accent: #ff72d0;

@mixin flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

⇒ **`vendors/`**

External imports:

- Bootstrap SCSS
- Swiper SCSS
- Normalize SCSS

⇒ **`base/`**

_Global style layer._

Contains:

- `_reset.scss` or `normalize`
- `_typography.scss`
- `_global.scss`

Example:

```scss
html {
  box-sizing: border-box;
}

body {
  font-family: "Inter", sans-serif;
  background: #fafafa;
}
```

⇒ **`layout/`**

Macro UI structure:

- `_header.scss`
- `_footer.scss`
- `_navbar.scss`
- `_sidebar.scss`
- `_grid.scss`

⇒ **`components/`**

Reusable UI blocks:

- `_buttons.scss`
- `_card.scss`
- `_modal.scss`
- `_form.scss`
- `_badge.scss`

Each component is fully modular—no global leaks.

⇒ **`pages/`**

If a page needs custom UI:

- `_home.scss`
- `_dashboard.scss`
- `_about.scss`

⇒ **`themes/`**

Multiple UI themes:

- `_light.scss`
- `_dark.scss`

### The Proper Import Order (IMPORTANT)

Import order defines the entire architecture.

`main.scss`:

```scss
// Abstracts (never generate CSS)
@import "abstracts/variables";
@import "abstracts/functions";
@import "abstracts/mixins";
@import "abstracts/breakpoints";

// Vendors (Bootstrap, Normalize, etc.)
@import "vendors/bootstrap";

// Base (global)
@import "base/reset";
@import "base/typography";
@import "base/global";

// Layout structure
@import "layout/header";
@import "layout/navbar";
@import "layout/footer";

// Components
@import "components/buttons";
@import "components/card";
@import "components/modal";

// Page styles
@import "pages/home";
@import "pages/dashboard";

// Themes (loaded last)
@import "themes/light";
@import "themes/dark";
```

Order rules:

1. **Variables**, **mixins**, **functions** first
2. **Vendor frameworks** second
3. **Global styles**
4. **Layout**, then **components** and **pages**
5. **Theme** overrides last (so they win)

### Modular Component Pattern

Each UI component should be self-contained.

Example file: `components/_card.scss`

```scss
.card {
  background: white;
  border-radius: $radius-lg;
  padding: $spacer * 2;
  box-shadow: $shadow-md;

  &--primary {
    border-left: 5px solid $primary;
  }

  &__title {
    font-size: 1.2rem;
    margin-bottom: 0.5rem;
  }

  &__content {
    color: $text-muted;
  }
}
```

Explanation:

- `&--primary { ... }`, because of `&`, it becomes, `.card--primary { ... }`
- `&__title` compiles to `.card__title`

### Global Theme System

A theme file only overrides variables.

Example: `themes/_dark.scss`

```scss
[data-theme="dark"] {
  --bg: #121212;
  --bg-card: #1e1e1e;
  --text: #e0e0e0;
  --text-muted: #aaaaaa;

  background-color: var(--bg);
  color: var(--text);

  .card {
    background-color: var(--bg-card);
  }
}
```

Switch theme using JS:

```js
document.documentElement.dataset.theme =
  document.documentElement.dataset.theme === "dark" ? "light" : "dark";
```

### Responsive Architecture

Use one global breakpoint file:

`abstracts/_breakpoints.scss`:

```scss
$breakpoints: (
  "sm": 576px,
  "md": 768px,
  "lg": 992px,
  "xl": 1200px,
);

@mixin respond($point) {
  @media (min-width: map-get($breakpoints, $point)) {
    @content;
  }
}
```

Usage:

```scss
.card {
  padding: 1rem;

  @include respond("lg") {
    padding: 2rem;
  }
}
```

### Utility Layer (like Tailwind-lite)

You can generate atomic utilities.

`utilities/_spacing.scss`:

```scss
@for $i from 1 through 5 {
  .m-#{$i} {
    margin: $i * 0.25rem;
  }
  .p-#{$i} {
    padding: $i * 0.25rem;
  }
}
```

## Lesson 3 - Built-In Functions [⬆](#)

Color, String, Number, List, Map

### COLOR FUNCTIONS (MOST USED IN REAL PROJECTS)

⇒ `lighten()` / `darken()`: Used to adjust color brightness. Essential for theme systems

```scss
$primary: #4a72ff;

.btn {
  background: $primary;
  &:hover {
    background: darken($primary, 10%); // dark variant
  }
}

.card {
  border-color: lighten($primary, 25%); // lighter border
}
```

⇒ `mix()`: Blend two colors

```scss
$primary: #4a72ff;
$black: #000;

.shadow-color: mix($primary, $black, 30%); // 30% primary + 70% black
```

⇒ `rgba()`: Add transparency

```scss
background: rgba($primary, 0.2);
```

⇒ `adjust-hue()`: Rotate hue angle to create color palettes

```scss
$new-color: adjust-hue($primary, 45deg);
```

⇒ `saturate()` / `desaturate()`: Control color intensity

```scss
tag {
  color: saturate($primary, 20%);
}
```

### NUMBER FUNCTIONS (MATH OPERATIONS)

⇒ Arithmetic

```scss
$base: 16px;
$large: $base * 2; // 32px
```

⇒ Percentage conversion

```scss
$ratio: percentage(0.25); // 25%
```

⇒ `min()` / `max()` / `clamp()`

```scss
title {
  font-size: clamp(1rem, 2vw, 2rem);
}
```

Note: Useful for responsive typography.

### STRING FUNCTIONS

⇒ `quote()` / `unquote()`

```scss
$name: unquote("primary"); // primary
```

⇒ `str-insert` / `str-replace` (Sass only)

```scss
$str: "btn-primary";
$new: str-replace($str, "primary", "secondary"); // btn-secondary
```

### LIST FUNCTIONS (CRITICAL FOR UTILITY GENERATION)

Lists are like arrays

⇒ `nth(list, index)`

```scss
$space-scale: (0, 4px, 8px, 16px, 32px);
margin: nth($space-scale, 3); // 8px
```

⇒ `append()` / `join()`

```scss
$list: (red, blue);
$new-list: append($list, green); // (red, blue, green)
```

### MAP FUNCTIONS (MOST IMPORTANT FOR THEME SYSTEMS)

Maps behave like JavaScript objects.

⇒ `map-get()`

```scss
$colors: (
  "primary": #4a72ff,
  "danger": #ff4757,
);

.btn {
  background: map-get($colors, "primary");
}
```

⇒ `map-merge()`

Add or override keys.

```scss
$extended: map-merge(
  $colors,
  (
    "success": #28a745,
  )
);
```

⇒ `map-remove()`

```scss
$no-danger: map-remove($colors, "danger");
```

### ADVANCED PRACTICAL EXAMPLE (IMPORTANT)

This example shows real production usage.

Color System

```scss
$colors: (
  "primary": #4a72ff,
  "primary-light": lighten(#4a72ff, 10%),
  "primary-dark": darken(#4a72ff, 10%),
  "danger": #ff4757,
  "success": #28a745,
);

@each $name, $value in $colors {
  .text-#{$name} {
    color: $value;
  }
  .bg-#{$name} {
    background-color: $value;
  }
}
```

Generates utilities like: `.text-primary`, `.bg-primary-dark`, `.text-success`, etc.

Exactly like Bootstrap’s internal system.

## Lesson 4 - Core Logic Layer [⬆](#)

Operators, Conditionals, Loops, Mixins & Functions

### OPERATORS (Math + Comparison)

SCSS supports arithmetic and logical comparison operators.

**Arithmetic**

```scss
$base: 1rem;
$lg: $base * 2; // 2rem
$half: $base / 2; // 0.5rem
$pad: 10px + 5px; // 15px
```

**Comparison**

```scss
$size: 12;

@if $size > 10 {
  font-size: 1.2rem;
}
```

**String Interpolation**

Important for generating class names.

```scss
$color: "primary";

.btn-#{$color} {
  color: blue;
}
```

### CONDITIONALS (`@if`, `@else if`, `@else`)

Allows branching logic.

Example: **Simple conditional**

```scss
$theme: "dark";

@if $theme == "dark" {
  background: #000;
  color: #fff;
} @else if $theme == "light" {
  background: #fff;
  color: #000;
} @else {
  background: #f5f5f5;
}
```

Use case: **Dynamic radius**

```scss
$shape: "rounded";

div {
  @if $shape == "circle" {
    border-radius: 50%;
  } @else if $shape == "rounded" {
    border-radius: 10px;
  } @else {
    border-radius: 0;
  }
}
```

### LOOPS (`@for`, `@each`, `@while`)

Loops are essential for generating frameworks (like Bootstrap, Tailwind, UIKit).

**`@for` loop (index loop)**

    <span style="font-style: italic;">Syntax:</span>

```scss
@for $i from 1 through 5 {
  .m-#{$i} {
    margin: $i * 4px;
  }

```

**`@each` loop (map or list iteration)**

_Example with list_:

```scss
$colors: red, green, blue;

@each $c in $colors {
  .text-#{$c} {
    color: $c;
  }
}
```

_Example with map_:

```scss
$theme-colors: (
  "primary": #4a72ff,
  "danger": #ff4757,
  "success": #28a745,
);

@each $name, $value in $theme-colors {
  .bg-#{$name} {
    background-color: $value;
  }
}
```

**`@while` loop**

Rarely used, but available.

```scss
$i: 1;

@while $i < 4 {
  .p-#{$i} {
    padding: $i * 6px;
  }
  $i: $i + 1;
}
```

### MIXINS (`@mixin` + `@include`)

Mixins = reusable styling blocks with optional parameters.

**Basic mixin**

```scss
@mixin flex-center {
  display: flex;
  align-items: center;
  justify-content: center;
}

.box {
  @include flex-center;
}
```

**Mixin with parameters**

```scss
@mixin size($w, $h: $w) {
  width: $w;
  height: $h;
}

.avatar {
  @include size(40px);
}
```

**Mixin for responsive breakpoints**

This is how Bootstrap works.

```scss
$breakpoints: (
  "sm": 576px,
  "md": 768px,
  "lg": 992px,
  "xl": 1200px,
);

@mixin respond($bp) {
  @media (min-width: map-get($breakpoints, $bp)) {
    @content; // Allows mixin to wrap code
  }
}

.card {
  padding: 12px;

  @include respond("md") {
    padding: 24px;
  }
}
```

### FUNCTIONS (`@function`)

Functions return values.

Essential for building tokens, design systems, utility engines.

**Basic function**

```scss
@function spacing($step) {
  @return $step * 4px;
}

.card {
  padding: spacing(3); // 12px
}
```

**Function with logic**

```scss
@function color-shade($color, $variant) {
  @if $variant == "light" {
    @return lighten($color, 15%);
  } @else if $variant == "dark" {
    @return darken($color, 15%);
  } @else {
    @return $color;
  }
}

.btn-primary {
  background: color-shade(#4a72ff, "dark");
}
```

### ADVANCED PRACTICAL SYSTEM (REAL PROJECT LEVEL)

Below is how modern frameworks auto-generate utilities.

Spacing scale + loop generation

```scss
$space-scale: (
  1: 4px,
  2: 8px,
  3: 12px,
  4: 16px,
  5: 24px,

@each $key, $val in $space-scale {

    margin: $val;
  }
  .p-#{$key} {
    padding: $val;
  }
  .mt-#{$key} {
    margin-top: $val;
  }
}
```

Color theme auto-generation

```scss
$colors: (
  "primary": #4a72ff,
  "danger": #ff4757,
  "success": #28a745,
);

@each $name, $color in $colors {
  .text-#{$name} {
    color: $color;
  }
  .bg-#{$name} {
    background-color: $color;
  }
  .border-#{$name} {
    border-color: $color;
  }
}
```
