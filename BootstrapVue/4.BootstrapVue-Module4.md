# Module 4 — Navigation & Structure [⬆](./0.BootstrapVue-doc.md)

## Navbar (Primary Navigation System)

A navbar:

- Provides **global navigation**
- Must work on **mobile first**
- Must be **keyboard and screen-reader accessible**
- Must not break layout flow

BootstrapVue’s `<b-navbar>` abstracts:

- Flex layout
- Toggle logic
- ARIA attributes
- Responsive collapse

### Minimal Correct Navbar

```vue
<b-navbar toggleable="lg" type="light" variant="light">
  <b-navbar-brand href="/">
    MyApp
  </b-navbar-brand>

  <b-navbar-toggle target="nav-collapse" />

  <b-collapse id="nav-collapse" is-nav>
    <b-navbar-nav>
      <b-nav-item href="/">Home</b-nav-item>
      <b-nav-item href="/about">About</b-nav-item>
    </b-navbar-nav>
  </b-collapse>
</b-navbar>
```

**Breakdown**:

- `toggleable="lg"` → collapses below large screens
- `b-navbar-toggle` → hamburger button
- `b-collapse` → controlled visibility
- `is-nav` → nav-specific styling

### Responsive Behavior (Critical)

`toggleable="lg"` means:

- Desktop (≥992px): always expanded
- Mobile/tablet: collapsed behind toggle

This is **mobile-first** design.

### Navbar Variants & Types

```vue
<b-navbar type="dark" variant="dark"></b-navbar>
```

- `type` controls text/icon color
- `variant` controls background

Should be always paired correctly:

- `dark + dark`
- `light + light`

Incorrect pairing causes contrast issues.

### Router Integration (Correct Way)

If using Vue Router:

```vue
<b-nav-item to="/" exact>
  Home
</b-nav-item>
```

BootstrapVue automatically uses `<router-link>`.

Benefits:

- Active link styling
- SPA navigation
- Accessibility preserved

## Secondary Navigation (Navs, Tabs & Pills)

### Nav vs Navbar (Important Distinction)

| Component | Purpose                  |
| --------- | ------------------------ |
| Navbar    | Global site navigation   |
| Nav       | Section-level navigation |
| Tabs      | Content switching        |
| Pills     | Alternate tab style      |

### Basic Nav

```vue
<b-nav>
  <b-nav-item active>Profile</b-nav-item>
  <b-nav-item>Settings</b-nav-item>
  <b-nav-item disabled>Billing</b-nav-item>
</b-nav>
```

Used for:

- Side menus
- Simple navigation lists

### Tabs (Controlled Content)

```vue
<b-tabs>
  <b-tab title="Overview">
    Overview content
  </b-tab>

  <b-tab title="Details">
    Details content
  </b-tab>
</b-tabs>
```

Tabs:

- Only one panel visible at a time
- Maintain internal state

### Lazy Loading Tabs (Performance)

```vue
<b-tab title="Reports" lazy>
  Heavy content loads only when clicked
</b-tab>
```

`lazy` used for:

- API calls
- Heavy charts
- Large tables

### Pills (Alternate Style)

```vue
<b-nav pills>
  <b-nav-item active>Day</b-nav-item>
  <b-nav-item>Week</b-nav-item>
  <b-nav-item>Month</b-nav-item>
</b-nav>
```

Pills used for:

- Filters
- Mode switches
- Non-hierarchical options

## Breadcrumbs (Context Awareness)

Breadcrumbs answer:

> “Where am I in the application?”

### Basic Breadcrumb

```vue
<b-breadcrumb>
  <b-breadcrumb-item href="/">Home</b-breadcrumb-item>
  <b-breadcrumb-item href="/blog">Blog</b-breadcrumb-item>
  <b-breadcrumb-item active>
    Post Title
  </b-breadcrumb-item>
</b-breadcrumb>
```

Rules:

- Last item is always `active`
- Breadcrumbs are informational, not primary nav

### Router-Based Breadcrumbs

```vue
<b-breadcrumb-item to="/dashboard">
  Dashboard
</b-breadcrumb-item>
```

Supports Vue Router seamlessly.
