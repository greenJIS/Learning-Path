## BootstrapVue Directives

### Components vs Directives

| Aspect            | Component | Directive    |
| ----------------- | --------- | ------------ |
| Renders markup    | Yes       | No           |
| Adds behavior     | Yes       | Yes          |
| Affects structure | Yes       | No           |
| Lightweight       | Medium    | Very         |
| Best for          | UI blocks | Enhancements |

Rule of thumb:

- for **markup** â†’ component
- for **behavior** â†’ directive

Directives Matter because Directives allow us to:

- Enhance existing elements
- Avoid unnecessary wrapper components
- Improve performance
- Keep templates readable

They follow Vueâ€™s `v-*` syntax and integrate seamlessly.

### `v-b-tooltip` (Micro-Help Done Right)

When to Use Tooltips

- Icon-only buttons
- Dense interfaces
- Secondary clarification

**Basic Tooltip**

```vue
<b-button v-b-tooltip.hover title="Delete item">
  ðŸ—‘
</b-button>
```

Key points:

- Tooltip text comes from `title`
- `.hover` defines trigger
- No extra markup created

**Trigger Variants**

- `v-b-tooltip.click`
- `v-b-tooltip.focus`
- `v-b-tooltip.hover.focus`

Use `focus` for keyboard accessibility.

**Placement**

- `v-b-tooltip.right`
- `v-b-tooltip.bottom`

Always test on small screens.

## `v-b-popover` (Richer Context)

### Tooltip vs Popover

| Tooltip    | Popover            |
| ---------- | ------------------ |
| Short text | Rich content       |
| One line   | Multiple sentences |
| Passive    | Informational      |

**Basic Popover**

```vue
<b-button
  v-b-popover.hover
  title="Warning"
  content="This action cannot be undone."
>
  Details
</b-button>
```

Use popovers for explanations, not instructions.

**Trigger Control**

```vue
v-b-popover.click
```

Prefer click for mobile compatibility.

## `v-b-toggle` (Visibility Control)

This directive connects triggers to collapsible content.

**Basic Toggle**

```vue
<b-button v-b-toggle.details>
  Toggle Details
</b-button>

<b-collapse id="details">
  Hidden content
</b-collapse>
```

No JS required.

**Multiple Triggers**

```vue
<b-button v-b-toggle.panel />
<a v-b-toggle.panel>Open</a>
```

Multiple triggers can control the same collapse.

## `v-b-scrollspy` (Advanced)

Scrollspy updates navigation based on scroll position.

Use Case

- Long documentation pages
- Section-based navigation
- Single-page scrolling layouts

**Basic Usage**

```vue
<b-nav v-b-scrollspy>
  <b-nav-item href="#section1">Section 1</b-nav-item>
  <b-nav-item href="#section2">Section 2</b-nav-item>
</b-nav>
```

Target sections must have IDs.

## Architectural Decision Guide

**Use Directives When:**

- Enhancing existing elements
- Behavior is simple
- Performance matters
- Markup must remain unchanged

**Avoid Directives When:**

- UI structure changes
- State is complex
- Reusability is required
