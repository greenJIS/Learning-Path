## Bootstrap Grid Fundamentals

### The Grid Mental Model

Bootstrap uses a **12-column grid system**.

Think of every row as:

```
| 1 | 2 | 3 | ... | 12 |
```

Columns **share** these 12 units.

Examples:

- 1 column full width → 12
- 2 equal columns → 6 + 6
- 3 equal columns → 4 + 4 + 4

If sizes are not specified, Bootstrap **automatically distributes space evenly**.

### The Three Core Layout Components

1️⃣ `<b-container>`

Purpose:

- Provides horizontal padding
- Centers content
- Defines max-width at each breakpoint

Types:

- `<b-container>` → fixed width (responsive)
- `<b-container fluid>` → full width (edge to edge)

2️⃣ `<b-row>`

Purpose:

- Creates a horizontal flex container
- Resets column padding via negative margins
- Ensures columns align correctly

Rule:

> Columns must always be children of rows.

3️⃣ `<b-col>`

Purpose:

- Represents a column
- Uses flexbox under the hood
- Automatically responsive

Example:

```vue
<b-container>
  <b-row>
    <b-col>Column A</b-col>
    <b-col>Column B</b-col>
    <b-col>Column C</b-col>
  </b-row>
</b-container>
```

**Bootstrap**:

- Detects 3 columns
- Divides 12 by 3
- Assigns `col-4` to each

This is **implicit sizing**.

**Explicit Column Sizing**: Control column width can be controled using `cols`.

```vue
<b-row>
  <b-col cols="8">Main Content</b-col>
  <b-col cols="4">Sidebar</b-col>
</b-row>
```

Rule:

> Column sizes in a row must add up to **≤ 12**

If they exceed 12, columns wrap to the next line.

### Responsive Breakpoints (Critical)

Bootstrap has predefined breakpoints:

| Breakpoint | Width   |
| :--------: | ------- |
|    `xs`    | <576px  |
|    `sm`    | ≥576px  |
|    `md`    | ≥768px  |
|    `lg`    | ≥992px  |
|    `xl`    | ≥1200px |

In BootstrapVue, you express this directly on `<b-col>`.

Example: Responsive Sidebar

```vue
<b-col cols="12" md="4" lg="3">
  Sidebar
</b-col>

<b-col cols="12" md="8" lg="9">
  Content
</b-col>
```

Meaning:

- Mobile: stacked (12 + 12)
- Tablet: 4 / 8
- Desktop: 3 / 9

**Visualizing Layout Before Coding**

Before writing code, some questions must be asked:

1. How many columns?
2. How do they behave on mobile?
3. When do they stop stacking?

This mindset prevents layout bugs.

### Hands-On Exercise

Building layout:

**Mobile**

```css
[ Header ]
[ Content ]
[ Sidebar ]
```

**Desktop**

```pgsql
[ Header Header Header ]
[ Content Content Sidebar ]
```

Solution

```vue
<b-container>
  <b-row>
    <b-col cols="12">
      Header
    </b-col>
  </b-row>

  <b-row>
    <b-col cols="12" lg="8">
      Content
    </b-col>
    <b-col cols="12" lg="4">
      Sidebar
    </b-col>
  </b-row>
</b-container>
```

## Advanced Grid Control

### Column Ordering

Visual order can be changed without changing markup.

```vue
<b-col cols="12" md="8" order="2" order-md="1">
  Main Content
</b-col>

<b-col cols="12" md="4" order="1" order-md="2">
  Sidebar
</b-col>
```

Rule:

> Mobile-first layouts<br>Accessibility-friendly markup

### Offsets (Spacing Columns)

Offsets shift columns to the right.

```vue
<b-col cols="6" offset="3">
  Centered Column
</b-col>
```

This creates symmetrical layouts without empty columns.

### Vertical Alignment

Rows are flex containers.

```vue
<b-row align-v="center">
  <b-col>Left</b-col>
  <b-col>Right</b-col>
</b-row>
```

Example values: `start`, `center`, `end`, `stretch`

### Nested Grids

Columns can contain rows.

```vue-demo
<b-col cols="8">
  <b-row>
    <b-col cols="6">Inner A</b-col>
    <b-col cols="6">Inner B</b-col>
  </b-row>
</b-col>
```

Rule:

> Container → Row → Column → Row → Column

Level must not be skiped

## Utility-First Layout Design

Bootstrap utilities:

- Are responsive
- Are standardized
- Reduce CSS debt
- Improve consistency

### Spacing Utilities

Examples:

- `p-3` → padding
- `mt-4` → margin-top
- `px-lg-5` → responsive padding

```vue
<b-col class="p-4 mt-3">
  Content
</b-col>
```

### Flex Utilities

```vue
<div class="d-flex justify-content-between align-items-center">
  <span>Left</span>
  <span>Right</span>
</div>
```

These utility classes can be applied instead of writing CSS flex rules

### Responsive Display

```vue
<div class="d-none d-md-block">
  Desktop Only
</div>
```

This will display this div on DOM only when screen size is `md` (≥768px)
