## Buttons & Button Groups

### Button Variants (Semantic Meaning)

Variants are **not colors**. They represent **intent**.

Common variants:

- <span style="color:#007bff;">primary → main action</span><br>
- <span style="color:#6c757d;">secondary → supporting action</span><br>
- <span style="color:#28a745;">success → positive confirmation</span><br>
- <span style="color:#dc3545;">danger → destructive action</span><br>
- <span style="color:#ffc107;">warning → caution</span><br>
- <span style="color:#17a2b8;">info → informational</span><br>
- <span style="color:#f8f9fa; background:#343a40; padding:2px 6px;">light → neutral UI</span><br>
- <span style="color:#343a40; background:#f8f9fa; padding:2px 6px;">dark → neutral UI</span>

Example:

```vue
<b-button variant="danger">
  Delete Account
</b-button>
```

> Use variants semantically, not aesthetically.

### Button Sizes & Block Buttons

```vue
<b-button size="sm">Small</b-button>
<b-button>Default</b-button>
<b-button size="lg">Large</b-button>
```

Full-width (block):

```vue
<b-button block variant="primary">
  Continue
</b-button>
```

Use `block` for:

- Mobile-first forms
- Primary CTA buttons

### Disabled & Loading States (Critical UX)

**Disabled Button**

```vue
<b-button :disabled="isSaving">
  Save
</b-button>
```

When disabled:

- Button is non-interactive
- Screen readers are informed
- Click events do not fire

**Loading Pattern (Best Practice)**

BootstrapVue does **not** have a built-in loading button, so you control it:

```vue
<b-button :disabled="loading" variant="primary">
  <b-spinner small v-if="loading" class="mr-2" />
  {{ loading ? 'Saving...' : 'Save' }}
</b-button>
```

This pattern:

- Prevents double submission
- Communicates progress
- Is accessible

### Button Events

Buttons emit standard Vue events:

```vue
<b-button @click="submitForm">
  Submit
</b-button>
```

Inline JS logic should never be used in templates (Always call methods)

### Buttons as Links (Correct Way)

If a button navigates instead of performing an action, it should be a **link**.

```vue
<b-button href="/pricing" variant="outline-primary">
  View Pricing
</b-button>
```

Or with Vue Router:

```vue
<b-button to="/pricing" variant="outline-primary">
  View Pricing
</b-button>
```

BootstrapVue automatically renders:

- `<a>` for links
- `<router-link>` for routes

This preserves semantics and accessibility.

## Cards (Content Containers)

Cards are the **most used structural component** in real applications.

A card is:

- A **content boundary**
- A visual grouping mechanism
- A flexible container using slots

Use cards to represent:

- Articles
- Products
- Dashboards widgets
- Profiles

### Basic Card Structure

```vue
<b-card title="Card Title">
  This is card content.
</b-card>
```

Internally:

- Header → title
- Body → slot content

### Header, Footer & Slots (Important)

```vue
<b-card>
  <template #header>
    Custom Header
  </template>

  Main content

  <template #footer>
    Footer content
  </template>
</b-card>
```

Slots used when **icons**, **multiple elements**, **full control** are needed

### Cards with Images

```vue
<b-card title="Blog Post" img-src="/cover.jpg" img-alt="Cover image" img-top>
  Article summary goes here.
</b-card>
```

> `img-alt` should always be provided for accessibility unless image is for decoration

### Card Layouts (Groups & Decks)

```vue
<b-card-group deck>
  <b-card title="One" />
  <b-card title="Two" />
  <b-card title="Three" />
</b-card-group>
```

Used for:

- Feature lists
- Pricing plans
- Product grids

## Alerts, Badges & Progress

These are used for **status communication**.

### Alerts (User Feedback)

```vue
<b-alert variant="success" show>
  Profile updated successfully.
</b-alert>
```

Dismissible alert:

```vue
<b-alert variant="warning" dismissible @dismissed="onDismiss">
  Session expired.
</b-alert>
```

Alerts used for:

- System messages
- Form feedback
- Global notices

### Badges (Labels, Not Decorations)

```vue
<b-badge variant="info">
  New
</b-badge>
```

Attached to content:

```vue
<h5>
  Messages
  <b-badge variant="danger">3</b-badge>
</h5>
```

Badges communicate **state**, not style.

### Progress Bars

```vue
<b-progress :value="70" max="100" />
```

With labels:

```vue
<b-progress :value="uploadProgress" show-progress />
```

Progress bars used for:

- Uploads
- Steps
- Completion indicators
