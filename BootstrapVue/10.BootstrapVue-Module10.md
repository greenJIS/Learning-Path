## Styling Strategy (Before Touching CSS)

### BootstrapVue Styling Layers (Mental Model)

There are **four layers**, in this order:

1. **Bootstrap SCSS variables**
2. **Bootstrap component styles**
3. **BootstrapVue component wrappers**
4. **Your custom styles**

If you style at the wrong layer, you lose control.

### Golden Rule of Customization

> Change variables first, override styles last

Why:

- Variables scale
- Overrides rot
- Specificity wars kill maintainability

## Using Bootstrap Utility Classes (Preferred)

Before writing CSS, ask:

> Can utilities solve this?

### Common Utilities You Should Prefer

- Spacing: `m-`, `p-`
- Layout: `d-flex`, `align-items-`, `justify-content-`
- Text: `text-`, `font-weight-`
- Visibility: `d-none`, `d-md-block`

Example:

```vue
<b-card class="p-4 d-flex justify-content-between align-items-center"></b-card>
```

No CSS needed.

**Why Utilities Are Safe**

- Stable API
- Responsive-ready
- No selector conflicts
- Self-documenting markup

## Variant System (Colors Done Right)

BootstrapVue uses semantic variants, not raw colors.

### Built-in Variants

<span style="color:#0d6efd">Primary</span>  
<span style="color:#6c757d">Secondary</span>  
<span style="color:#198754">Success</span>  
<span style="color:#dc3545">Danger</span>  
<span style="color:#ffc107">Warning</span>  
<span style="color:#0dcaf0">Info</span>  
<span style="color:#f8f9fa; background:#000; padding:2px 6px;">Light</span>  
<span style="color:#212529; background:#fff; padding:2px 6px;">Dark</span>

Use meaning, not aesthetics.

### Variants in Components

```vue
<b-button variant="success" />
<b-alert variant="danger" />
<b-badge variant="info" />
```

Variants propagate consistently across components.

### Custom Variants (Advanced)

Define in SCSS:

```scss
$theme-colors: (
  "primary": #6f42c1,
  "brand": #ff5722,
);
```

Then rebuild Bootstrap.

Avoid ad-hoc colors in components.

## Scoped CSS vs Global CSS (Critical Choice)

### Scoped CSS (When to Use)

Use scoped CSS when:

- Styling a single component
- Behavior is local
- No Bootstrap internals are touched

```vue
<style scoped>
.card-title {
  font-weight: 600;
}
</style>
```

### Global CSS (When Required)

Use global CSS when:

- Modifying Bootstrap components
- Enforcing brand rules
- Styling shared patterns

```scss
.btn {
  border-radius: 0.75rem;
}
```

Put this in a global stylesheet, not scoped.

### Scoped CSS Gotcha

Scoped styles **do not penetrate child components** reliably.

Never rely on them for framework overrides.

## Customizing BootstrapVue Components Safely

### Never Target Internal Markup Blindly

❌ Bad:

```css
.b-table tbody tr td:nth-child(3) {
  color: red;
}
```

Breaks on updates.

### Preferred Pattern: Wrapper Classes

```vue
<b-table class="user-table" />
```

```scss
.user-table td {
  vertical-align: middle;
}
```

Always anchor styles to **your own class**.

## Dark Mode Strategy

### BootstrapVue Reality

BootstrapVue (v2) is built on Bootstrap 4 → **no native dark mode**.

You must:

- Control variables
- Or maintain dual themes

### Practical Dark Mode Pattern

- Use CSS variables
- Toggle root class

```css
:root {
  --bg: #ffffff;
  --text: #212529;
}

.dark {
  --bg: #121212;
  --text: #eaeaea;
}
```

Use variables inside custom styles, not Bootstrap internals.
