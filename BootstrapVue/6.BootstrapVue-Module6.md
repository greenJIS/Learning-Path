## Modals (Dialogs Done Right)

**When to use (and Not to) a Modal**

‚úÖ to use a modal when:

- User must make a focused decision
- Content is short and self-contained
- You must block background interaction temporarily

‚ùå to use a modal when:

- Content is long or scroll-heavy
- Navigation is required
- User needs context from the page

Modals are **interruptions**. Use them intentionally.

### Basic Modal Structure

```vue
<b-modal id="confirm-modal" title="Confirm Action">
  Are you sure you want to continue?

  <template #modal-footer>
    <b-button variant="secondary" @click="$bvModal.hide('confirm-modal')">
      Cancel
    </b-button>
    <b-button variant="danger">
      Confirm
    </b-button>
  </template>
</b-modal>
```

Trigger:

```vue
<b-button v-b-modal.confirm-modal>
  Open Modal
</b-button>
```

Key ideas:

- Modal is declared once
- Triggered from anywhere
- ID-based control

### Programmatic Control (Best Practice)

```js
this.$bvModal.show("confirm-modal");
this.$bvModal.hide("confirm-modal");
```

Programmatic control used when:

- Modal depends on data
- Confirmation affects logic
- You want clean templates

# Preventing Accidental Close

```vue
<b-modal id="safe-modal" no-close-on-backdrop no-close-on-esc />
```

Use this for:

- Critical forms
- Unsaved changes
- Destructive actions

### Modal Lifecycle Events

```vue
<b-modal @show="onShow" @shown="onShown" @hide="onHide" @hidden="onHidden" />
```

Use cases:

- Reset form on `show`
- Focus input on `shown`
- Confirm close on `hide`

## Toasts (Non-Intrusive Feedback)

Toasts communicate **status without blocking flow**.

Use toasts for:

- Success messages
- Background operations
- Non-critical errors

Do NOT use toasts for:

- Forms requiring correction
- Critical blocking issues

### Basic Toast Usage

```js
this.$bvToast.toast("Profile saved successfully.", {
  title: "Success",
  variant: "success",
  solid: true,
});
```

Key options:

- `title`
- `variant`
- `autoHideDelay`
- `solid`

### Toast Positioning

```js
this.$bvToast.toast("Hello", {
  toaster: "b-toaster-top-right",
});
```

Standard positions:

- top-right
- top-center
- bottom-right
- bottom-left

Choose one globally and stay consistent.

### Toasts in Async Flows

```js
async save() {
  try {
    await api.save()
    this.$bvToast.toast('Saved!', { variant: 'success' })
  } catch {
    this.$bvToast.toast('Failed', { variant: 'danger' })
  }
}
```

## Tooltips & Popovers

These add contextual help without clutter.

### Tooltips (Micro-Help)

```vue

  üóë
</b-button>
```

Use tooltips for:

- Icon-only buttons
- Abbreviations
- Clarifying actions

### Popovers (Richer Context)

```vue
<b-button
  v-b-popover.hover
  title="Details"
  content="This action cannot be undone."
>
  Info
</b-button>
```

Use popovers when:

- More than one sentence
- Explanation required

### Accessibility Considerations

- Always provide a trigger
- Avoid hover-only for critical info
- Keep content short
