# Module 13 – Performance, Optimization & Best Practices [⬆](./0.BootstrapVue-doc.md)

## Rendering Strategy

### The Core Performance Problem

Most performance issues come from:

- Too many reactive bindings
- Over-rendered components
- Heavy DOM trees
- Poor state boundaries

BootstrapVue components are powerful—but **not free**.

### `v-if` vs `v-show`

**`v-if`**

- Removes element from DOM
- Expensive to toggle
- Best for rare states

```vue
<b-modal v-if="show" />
```

**`v-show`**

- Toggles CSS display
- Cheap to toggle
- Best for frequent visibility changes

```vue
<b-alert v-show="error" />
```

Rule

> Rare → `v-if`<br>Frequent → `v-show`

### Tables & Lists Optimization

**Problem: Large Tables**

`b-table` with hundreds of rows can be slow.

**Solutions**

1. Server-side pagination
2. Limit per-page rows
3. Disable unused features

```vue
<b-table :items="items" :per-page="20" no-local-sorting no-local-filtering />
```

Only enable what you use.

### Slot Discipline (Hidden Cost)

**Problem**

Slots rerender more than you expect.

**Best Practice**

- Keep slot templates simple
- Avoid inline methods
- Precompute values

❌ Bad:

```vue
<template #cell(name)="row">
  {{ formatName(row.item) }}
</template>
```

✅ Good:

```js
computed: {
  formattedItems() {
    return this.items.map(i => ({
      ...i,
      displayName: formatName(i)
    }))
  }
}
```

### Reactive Data Size Control

**Rule**

> Make data as small as possible, as late as possible

Avoid:

- Storing full API responses in reactive state
- Passing large objects through props

Extract only what the UI needs.

### Global Component Registration (Hidden Tax)

**Problem**

Registering everything globally increases:

- Bundle size
- Startup cost

**Recommendation**

Use **local registration** for heavy components:

```js
components: {
  BTable, BModal;
}
```

### CSS Performance

**Avoid**

- Deep selectors
- `*` selectors
- Large scoped styles

**Prefer**:

- Utility classes
- Shallow selectors
- Component-level classes

### Event Handling Discipline

**Anti-pattern**

```vue
<b-button @click="() => doThing(item)"></b-button>
```

Creates a new function every render.

**Correct Pattern**

```vue
<b-button @click="doThing(item)"></b-button>
```

Or pass IDs instead of objects.

### Production Checklist

Before shipping:

- Remove debug props
- Disable unused features
- Check bundle size
- Test large data sets
- Verify keyboard navigation
- Test on low-end devices

### BootstrapVue Lifecycle Reality

BootstrapVue v2:

- Built on Vue 2
- Bootstrap 4 based
- Maintenance mode

**Implication**

Use it for:

- Stable projects
- Internal tools
- Admin panels

Plan migration to:

- Vue 3

- Alternative UI libraries
