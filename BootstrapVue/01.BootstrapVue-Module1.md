# Module 1 - Environment & Setup [⬆](./0.BootstrapVue-doc.md)

## BootstrapVue Dependency

BootstrapVue is **not standalone**

It depends on **three layers**, all of which must be correct:

1. **Vue.js 2.6.x**
2. **Bootstrap v4 CSS**
3. **BootstrapVue component layer**

If any of these are missing or misordered, things will “work” but behave incorrectly.

### **Dependency Relationship**

```scss
Vue 2 (reactivity + templates)
   ↓
Bootstrap CSS (layout + styles)
   ↓
BootstrapVue (Vue components that emit Bootstrap markup)
```

BootstrapVue does **not** include Bootstrap CSS by default.

This is intentional to customize themes.

### **Version Compatibility (Critical)**

BootstrapVue is frozen and officially supports:

| Library      | Required Version |
| ------------ | ---------------- |
| Vue          | 2.6.x            |
| Bootstrap    | 4.x              |
| BootstrapVue | latest           |

❌ Vue 3 → NOT supported

❌ Bootstrap 5 → NOT supported

If Vue 3 is used, components may compile but fail at runtime.

### **Installing Dependencies**

**Using npm (recommended)**

```bash
npm install vue bootstrap@4 bootstrap-vue
```

Or with yarn:

```bash
yarn add vue bootstrap@4 bootstrap-vue
```

What this installs:

- `vue` → core framework
- `bootstrap@4` → CSS + utilities
- `bootstrap-vue` → Vue components

### **Correct Import Order**

on `mian.js`

❌ Wrong (Very Common)

```js
import "bootstrap-vue/dist/bootstrap-vue.css";
import "bootstrap/dist/css/bootstrap.css";
```

This breaks styles because BootstrapVue assumes Bootstrap is already loaded.

✅ Correct Order (Always)

```js
import "bootstrap/dist/css/bootstrap.css";
import "bootstrap-vue/dist/bootstrap-vue.css";
```

Rule:

> Bootstrap CSS **must load first**, BootstrapVue CSS second.

### **Plugin Registration (Global)**

**Minimal Correct Setup**

```js
import Vue from "vue";
import App from "./App.vue";

import { BootstrapVue, IconsPlugin } from "bootstrap-vue";

import "bootstrap/dist/css/bootstrap.css";
import "bootstrap-vue/dist/bootstrap-vue.css";

Vue.use(BootstrapVue);
Vue.use(IconsPlugin);

new Vue({
  render: (h) => h(App),
}).$mount("#app");
```

**What `Vue.use()` Does**

- Registers **all BootstrapVue components**
- Registers **all directives**
- Makes everything globally available

This is convenient for learning, but **not optimal for production**

Because this import: _80+ components_, _All directives_, _All utilities_. Even if only `<b-button>` is used.

This _Increases bundle size_, _Slows initial load_, _Makes audits harder_

**Solution**: Selective Component Import Strategy (**One component**, **One import**)

Example:

```js
import Vue from "vue";
import { BButton, BCard, BCardText } from "bootstrap-vue";

Vue.component("BButton", BButton);
Vue.component("BCard", BCard);
Vue.component("BCardText", BCardText);
```

**Best Practice (Hybrid Strategy)**

- Global plugin for layout & basics
- Selective imports for heavy components (tables, modals)

### **Common Failure Patterns (Real World)**

| Symptom                           | Cause                  |
| --------------------------------- | ---------------------- |
| Components render unstyled        | Bootstrap CSS missing  |
| Layout broken                     | CSS import order wrong |
| `<b-icon>` not found              | IconsPlugin missing    |
| Everything compiles but UI broken | Bootstrap 5 used       |
| Runtime warnings                  | Vue 3 used             |
