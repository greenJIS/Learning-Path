# Module 9 — Icons & Visual Consistency [⬆](./0.BootstrapVue-doc.md)

## BootstrapVue Icons System

BootstrapVue icons are:

- **Inline SVGs**
- Not fonts
- Not images

This gives you:

- Crisp rendering at any size
- CSS-based coloring
- Accessibility control
- No font-loading overhead

### Enabling Icons (Reminder)

Icons are **not included automatically**.

You must install and enable the plugin:

```js
import { IconsPlugin } from "bootstrap-vue";
Vue.use(IconsPlugin);
```

Without this, `<b-icon>` will not render.

### Basic Icon Usage

```vue
<b-icon icon="check-circle" />
```

Icon names match **Bootstrap Icons** naming.

Rule:

> If you know Bootstrap Icons, you know BootstrapVue icons.

### Icon Size Control

Icons scale via `font-size`, not width/height.

```vue
<b-icon icon="alarm" font-scale="1.5" />
<b-icon icon="alarm" font-scale="2" />
```

Use scale sparingly. Large icons should be intentional.

### Icon Color (Correct Way)

Icons inherit text color.

```vue
<b-icon icon="exclamation-triangle" variant="warning" />
```

Or via utility classes:

```vue
<b-icon icon="check" class="text-success" />
```

Avoid inline styles unless necessary.

## Icons with Buttons, Inputs & Navigation

### Icons Inside Buttons

```vue
<b-button variant="primary">
  <b-icon icon="save" class="mr-2" />
  Save
</b-button>
```

Rules:

- Icon comes before text for actions
- Use spacing utilities (`mr-2`)
- Never use icon-only buttons without a tooltip

### Icon-Only Buttons (Accessibility)

```vue
<b-button variant="danger" v-b-tooltip.hover title="Delete">
  <b-icon icon="trash" />
</b-button>
```

Required:

- Tooltip or `aria-label`
- Clear intent

Never rely on icon meaning alone.

### Icons in Form Inputs

```vue
<b-input-group>
  <b-input-group-prepend>
    <b-input-group-text>
      <b-icon icon="envelope" />
    </b-input-group-text>
  </b-input-group-prepend>

  <b-form-input />
</b-input-group>
```

Use icons in inputs only when they **add clarity**, not decoration.

## Icons in Tables, Lists & Status Indicators

Icons often replace text—this is dangerous if done poorly.

### Status Icons (Correct Pattern)

```vue
<template #cell(status)="row">
  <b-icon
    :icon="row.value ? 'check-circle' : 'x-circle'"
    :variant="row.value ? 'success' : 'danger'"
  />
</template>
```

Always pair:

- Icon
- Color
- Context (tooltip if needed)

### Avoid Icon Overload

❌ Bad:

- Multiple icons per cell
- No text fallback
- Meaning encoded only in color

✅ Good:

- One icon per concept
- Consistent meaning across app

## Visual Consistency Rules

### Icon Usage Ruleset

Adopt these rules in teams:

1. Same action → same icon everywhere
2. Same state → same icon + color
3. Icons supplement text, not replace it
4. Icon-only UI must be accessible

### Icon Sizing Discipline

- Body text icons: default size
- Buttons: match text size
- Feature icons: intentional scaling

Avoid random scaling.
